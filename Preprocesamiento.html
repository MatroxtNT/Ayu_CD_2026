1. ¿Qué es CRISP-DM?

CRISP-DM es un marco de trabajo utilizado para organizar proyectos de minería de datos, ciencia de datos y aprendizaje automático. Su función es ordenar el proceso completo, desde que se entiende el problema hasta que los resultados se interpretan o se aplican.

La idea principal es que un proyecto no comienza entrenando modelos, sino entendiendo primero el contexto, los datos y la calidad de la información disponible.

¿Para qué sirve?

Sirve para seguir un orden lógico en un proyecto de datos y no saltarse etapas importantes.

¿Qué organiza?

La comprensión del problema, el análisis de los datos, su preparación y el modelamiento posterior.

Idea clave

Primero se entiende el problema y los datos; después se transforman y recién luego se modela.

2. Etapa: Data Understanding

La etapa de Data Understanding busca conocer los datos antes de modificarlos. Aquí se revisa qué información existe, cómo está estructurada, qué tipos de variables aparecen, qué patrones se observan y si hay problemas de calidad.

2.1 Subetapa: Recolección inicial

En esta subetapa se reúnen los datos que estarán disponibles para el proyecto. La idea es identificar desde dónde vienen y en qué formato están.

Datos estructurados Datos semiestructurados Datos no estructurados

Datos estructurados

Son datos organizados en tablas con filas y columnas, como una base de datos o un archivo Excel.

Ejemplo: una tabla de estudiantes con columnas como ID, edad, carrera y promedio.

Datos semiestructurados

Tienen cierto orden, pero no siguen una tabla rígida. Suelen venir en formatos como JSON o XML.

Ejemplo: un archivo JSON con datos de usuarios y atributos variables.

Datos no estructurados

No están organizados en columnas fijas. Aquí suelen aparecer textos, imágenes, audios o videos.

Ejemplo: comentarios escritos por clientes o fotografías almacenadas en carpetas.

2.2 Subetapa: Descripción de los datos

En esta subetapa se explica qué representa cada atributo y qué tipo de dato contiene. Esto es importante porque el tipo de variable condiciona el análisis que se puede hacer después.

Numérica discreta

Corresponde a cantidades enteras y contables. No suele admitir valores intermedios con decimales.

Ejemplo: número de hijos, cantidad de cursos aprobados, total de productos.

Numérica continua

Puede tomar cualquier valor dentro de un rango, incluyendo decimales.

Ejemplo: promedio, peso, temperatura o ingreso mensual.

Categórica nominal

Representa categorías sin orden natural entre ellas.

Ejemplo: carrera, ciudad, color, tipo de contrato.

Categórica ordinal

Representa categorías que sí tienen un orden lógico entre sus niveles.

Ejemplo: bajo, medio y alto; o malo, regular y bueno.

2.3 Subetapa: Exploración de los datos

En la exploración de datos se revisan patrones iniciales, distribuciones, relaciones entre variables y comportamientos llamativos. Aquí se usan tablas resumen y gráficos para obtener una visión general.

En esta subetapa normalmente se hacen gráficos como histogramas, boxplots, gráficos de barras, gráficos de dispersión y mapas de calor.
Ejemplo: hacer un histograma para ver cómo se distribuyen los promedios o un gráfico de barras para comparar carreras.

2.4 Subetapa: Análisis de calidad

Aquí se detectan problemas que afectan la confiabilidad de los datos. Por ejemplo, valores faltantes, registros duplicados, errores de digitación, ruido, datos inconsistentes o valores extremadamente alejados.

La idea principal es revisar si los datos realmente son utilizables antes de pasar a la etapa de preparación.

3. Etapa: Data Preparation

La etapa de Data Preparation transforma los datos para dejarlos listos para análisis posteriores o para ser utilizados por un modelo. Aquí se seleccionan variables, se corrigen errores, se crean nuevas columnas y se transforman atributos.

3.1 Subetapa: Selección

La selección consiste en elegir qué datos, atributos o columnas serán útiles para el análisis. No siempre conviene trabajar con todo, porque puede haber variables irrelevantes, redundantes o poco informativas.

Ejemplo: si quiero analizar rendimiento académico, podría seleccionar promedio, asistencia y número de evaluaciones, y dejar fuera un atributo que no aporta.

3.2 Subetapa: Limpieza

La limpieza corrige problemas de calidad en los datos. Aquí se tratan registros repetidos, datos faltantes, valores atípicos y ruido.

Valores duplicados

Son registros repetidos total o parcialmente. Pueden distorsionar el análisis si se cuentan más de una vez.

Ejemplo: el estudiante con ID 123-k aparece dos veces con exactamente los mismos datos.
Qué se hace: normalmente se eliminan o se revisan para confirmar si realmente son duplicados.

Valores faltantes

Son datos ausentes, vacíos o no registrados en una celda.

Ejemplo: un estudiante tiene registrada su carrera y promedio, pero la columna de edad aparece vacía.
Qué se hace: se pueden eliminar, imputar con media, mediana, moda, o reemplazar según el contexto.

Outliers

Son valores muy alejados del comportamiento general de los datos.

Ejemplo: si la mayoría de las edades está entre 18 y 30 años, un registro de 70 años podría verse como un valor atípico.
Qué se hace: se revisan, se justifican, se corrigen o se eliminan si corresponden a errores o afectan el análisis.

Ruido

El ruido corresponde a errores o alteraciones que hacen que un dato no refleje correctamente la realidad.

Ejemplo: escribir “Bieenvenidp” en lugar de “Bienvenidos”, o digitar mal un número de teléfono.
Qué se hace: se detecta y se intenta corregir usando técnicas de comparación.

Detección matemática de outliers

Para detectar un outlier de forma más formal, se pueden usar medidas como Z-score o IQR.

Z-score

El Z-score mide cuántas desviaciones estándar se aleja un dato de la media. Si el valor absoluto del Z-score es muy alto, el dato podría considerarse atípico.

z = (x - media) / desviación_estándar
IQR

El IQR o rango intercuartílico mide la amplitud del 50% central de los datos. Un valor muy por debajo o muy por encima de ese rango puede ser un outlier.

IQR = Q3 - Q1 Límite inferior = Q1 - 1.5 * IQR Límite superior = Q3 + 1.5 * IQR

Ruido en datos: Hamming y Levenshtein

Dos medidas útiles para comparar cadenas o secuencias son la distancia de Hamming y la distancia de Levenshtein.

Distancia de Hamming

Cuenta cuántas posiciones distintas hay entre dos cadenas del mismo largo.

A = 8374629152 B = 8375628153

Comparando posición por posición:

  • Posición 4: 4 cambió por 5
  • Posición 7: 9 cambió por 8
  • Posición 10: 2 cambió por 3

Entonces la distancia de Hamming es 3, porque hay 3 posiciones diferentes.

Distancia de Levenshtein

Mide cuántas operaciones mínimas se necesitan para transformar una cadena en otra. Las operaciones son insertar, eliminar o sustituir caracteres.

Origen = Bieenvenidp Destino = Bienvenidos

Si queremos ir desde Bieenvenidp hasta Bienvenidos, una forma simple de verlo es esta:

  • Eliminar una e extra después de “Bi” para pasar de Bieenvenidp a Bienvenidp.
  • Sustituir la penúltima letra p por o para obtener Bienvenido.
  • Insertar la última letra s para obtener Bienvenidos.

Con esas 3 operaciones ya se logra la transformación, por lo que la distancia de Levenshtein es 3.

3.3 Subetapa: Construcción de datos

La construcción consiste en crear nuevas variables a partir de otras ya existentes. Esto permite representar mejor la información y generar atributos más útiles.

Ejemplo: si tengo Nota_1 y Nota_2, puedo construir una nueva columna llamada Promedio.
Promedio = (Nota_1 + Nota_2) / 2
La construcción agrega valor porque transforma datos originales en variables más interpretables.

3.4 Subetapa: Reconstrucción de datos

La reconstrucción transforma la forma en que se representan los datos. Aquí entran procesos como discretizar variables numéricas o cambiar la escala de los atributos.

3.4.1 Discretización

La discretización convierte una variable numérica continua en categorías o intervalos. Esto se usa cuando conviene pasar de números exactos a niveles como bajo, medio y alto.

Por amplitud

Divide el rango total en intervalos del mismo tamaño.

Ejemplo: si los promedios van de 4,0 a 7,0, se puede dividir en:
Bajo: 4,0 - 4,9
Medio: 5,0 - 5,9
Alto: 6,0 - 7,0
Por frecuencia

Divide los datos en grupos con cantidades similares de observaciones.

Ejemplo: si quiero tres categorías, cada grupo puede contener aproximadamente un tercio de los datos, aunque los intervalos no tengan el mismo ancho.

3.4.2 Escalamiento

El escalamiento consiste en cambiar la magnitud de las variables para que queden en rangos comparables. Esto es importante porque algunas variables pueden tener números mucho más grandes que otras y dominar los cálculos del modelo.

Ejemplo: una persona puede tener edad 20 e ingreso 800000, mientras otra tiene edad 30 e ingreso 1200000. La diferencia en ingreso parece enorme respecto de la edad, aunque ambas variables podrían ser importantes.
Escalar ayuda a que una variable no pese más que otra solamente por tener valores numéricos mayores.

3.4.3 Normalización

La normalización es una forma específica de escalamiento en la que los valores se llevan, por ejemplo, al rango [0,1]. Así todas las variables quedan en una escala comparable.

x_norm = (x - x_min) / (x_max - x_min)

Esta es la fórmula de Min-Max. Toma un valor original y lo convierte a un nuevo valor dentro del rango 0 a 1.

Variable Rango original Valor original Valor normalizado
Edad [18, 60] 20 (20 - 18) / (60 - 18) = 2 / 42 ≈ 0,047
Ingreso [300000, 1500000] 800000 (800000 - 300000) / (1500000 - 300000) = 500000 / 1200000 ≈ 0,416
Edad [18, 60] 30 (30 - 18) / (60 - 18) = 12 / 42 ≈ 0,285
Ingreso [300000, 1500000] 1200000 (1200000 - 300000) / (1500000 - 300000) = 900000 / 1200000 = 0,75
Interpretación: después de normalizar, la diferencia en edad y en ingreso ya puede compararse mejor, porque ambas quedaron dentro del mismo rango.

4. Resumen final

El marco CRISP-DM permite organizar un proyecto de datos paso a paso. En este contenido se revisaron dos etapas clave:

  • Data Understanding: entender qué datos existen, qué tipos tienen, cómo se comportan y qué problemas presentan.
  • Data Preparation: seleccionar, limpiar, construir y transformar datos para que queden listos para trabajar con ellos.
La idea central es que un buen análisis no depende solo del modelo, sino también de entender y preparar correctamente los datos desde el inicio.
