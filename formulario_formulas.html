1. Preprocesamiento

Fórmulas base para escalamiento, normalización, construcción de atributos y detección de outliers.

Min-Max Escalamiento Outliers Construcción

Normalización Min-Max

x_norm = (x - x_min) / (x_max - x_min)
Lleva un valor original a una escala comparable, normalmente entre 0 y 1.

Desnormalización Min-Max

x = x_norm * (x_max - x_min) + x_min
Permite recuperar el valor original a partir del valor normalizado.

Z-score

z = (x - media) / desviación_estándar
Mide cuántas desviaciones estándar se aleja un valor de la media.

IQR

IQR = Q3 - Q1 Límite inferior = Q1 - 1.5 * IQR Límite superior = Q3 + 1.5 * IQR
Método clásico para detectar valores atípicos usando cuartiles.

2. Ruido y comparación de cadenas

Medidas útiles para detectar diferencias entre secuencias o textos.

Distancia de Hamming

Hamming(A,B) = número de posiciones distintas
Compara posición por posición y cuenta cuántas son diferentes.

Ejemplo de Hamming

A = 8374629152 B = 8375628153 Cambios: posición 4: 4 → 5 posición 7: 9 → 8 posición 10: 2 → 3 Hamming(A,B) = 3
Solo se aplica cuando ambas cadenas tienen el mismo largo.

Levenshtein

Levenshtein(A,B) = mínimo número de inserciones + eliminaciones + sustituciones para transformar A en B
Se usa cuando las cadenas pueden tener distinto largo.

Idea rápida de Levenshtein

Si A = "canto" y B = "cuento" Levenshtein(A,B) = 2
Porque se requieren 2 operaciones mínimas para transformar una palabra en la otra.

3. Métricas de distancia

Fórmulas usadas especialmente en clustering y comparación de puntos.

Distancia Euclidiana

d(A,B) = √((x2 - x1)^2 + (y2 - y1)^2)
Distancia en línea recta. Muy usada en K-Means.

Distancia Manhattan

d(A,B) = |x2 - x1| + |y2 - y1|
Suma desplazamientos horizontales y verticales.

Distancia Chebyshev

d(A,B) = max(|x2 - x1|, |y2 - y1|)
Toma solamente la mayor diferencia entre dimensiones.

Distancia Minkowski

d(A,B) = (|x2 - x1|^p + |y2 - y1|^p)^(1/p)
Generaliza varias distancias. Si p = 1 es Manhattan, si p = 2 es Euclidiana.

4. K-Means

Algoritmo de clustering basado en centroides. Lo importante suele ser la asignación de puntos y el recálculo del centroide.

Asignación Centroide Iteración

4.1 Distancia de un punto a un centroide

Fórmula

Si el punto es P = (x_p, y_p) y el centroide es C = (x_c, y_c), entonces: d(P, C) = √((x_p - x_c)^2 + (y_p - y_c)^2)
Esta fórmula mide qué tan lejos está un punto de un centroide.

Ejemplo de asignación

P = (2, 3) C1 = (4, 5) C2 = (8, 3) d(P, C1) = √((2 - 4)^2 + (3 - 5)^2) d(P, C1) = √(4 + 4) d(P, C1) = √8 ≈ 2.83 d(P, C2) = √((2 - 8)^2 + (3 - 3)^2) d(P, C2) = √(36 + 0) d(P, C2) = 6 Como 2.83 < 6, el punto P se asigna a C1.
Regla general: el punto siempre se asigna al centroide más cercano.

4.2 Recalculo del centroide

Centroide en 2 dimensiones

C = ( (x1 + x2 + ... + xn) / n, (y1 + y2 + ... + yn) / n )
El centroide es el promedio de todos los puntos asignados al cluster.

Idea de iteración

1) Inicializar centroides 2) Asignar puntos 3) Recalcular centroides 4) Repetir hasta converger
Esa es la lógica principal del algoritmo.

5. DBSCAN

Algoritmo basado en densidad. Lo más importante aquí es entender el significado de epsilon y min points.

epsilon min points ruido densidad

Epsilon (eps)

eps = radio de vecindad
Define qué tan cerca deben estar los puntos para considerarse vecinos.

Min Points / Min Samples

MinPts = número mínimo de puntos necesarios dentro del radio eps
Sirve para decidir si una zona tiene suficiente densidad.

Interpretación de eps

eps muy pequeño → mucho ruido eps muy grande → clusters fusionados
Si el radio es demasiado chico, muchos puntos quedan aislados. Si es demasiado grande, varios grupos distintos pueden unirse.

Interpretación de MinPts

MinPts bajo → clusters se forman fácil MinPts alto → se exige mayor densidad
A mayor MinPts, más difícil es que una zona se considere cluster.

Idea general del algoritmo

1) Elegir eps 2) Elegir MinPts 3) Buscar zonas densas 4) Formar clusters 5) Dejar fuera el ruido
DBSCAN no usa centroides, usa densidad.

6. Reglas de asociación

Métricas fundamentales para interpretar reglas como X → Y.

Probabilidad básica

P(A) = casos favorables / casos totales
Base para entender soporte y probabilidades en reglas.

Probabilidad conjunta

P(X ∩ Y)
Probabilidad de que X e Y ocurran juntos.

Probabilidad condicional

P(Y|X) = P(X ∩ Y) / P(X)
Base directa para la confidence.

Bayes

P(X|Y) = (P(Y|X) * P(X)) / P(Y)
Sirve como base conceptual en probabilidad condicional.

Support

Support(X) = frecuencia(X) / total Support(X → Y) = Support(X ∩ Y)
Frecuencia relativa de un ítem o combinación en el dataset.

Confidence

Confidence(X → Y) = Support(X ∩ Y) / Support(X)
Indica qué tan frecuente es Y cuando aparece X.

Lift

Lift(X → Y) = Support(X ∩ Y) / (Support(X) * Support(Y))
Si lift = 1 hay independencia; si es mayor que 1, relación positiva.

Leverage

Leverage(X → Y) = Support(X ∩ Y) - (Support(X) * Support(Y))
Mide cuánto difiere la ocurrencia conjunta de lo esperado por azar.

Conviction

Conviction(X → Y) = (1 - Support(Y)) / (1 - Confidence(X → Y))
Otra medida para evaluar fuerza de una regla.

7. Apriori y Pruning

Apriori genera itemsets frecuentes y usa pruning para evitar revisar combinaciones innecesarias.

Itemsets Support mínimo Poda

Frecuencia de itemset

Support(A) = count(A) / N
Un itemset es frecuente si cumple el soporte mínimo.

Condición de frecuencia

A es frecuente si Support(A) ≥ min_support
Regla base para aceptar o rechazar itemsets.

Principio Apriori

Si un itemset es frecuente, entonces todos sus subconjuntos también deben ser frecuentes.
Esta es la idea que permite hacer pruning.

Pruning

Si existe un subconjunto de A que no es frecuente, entonces A no puede ser frecuente.
Si un subconjunto falla, se poda el conjunto mayor completo.

Ejemplo simple de pruning

Queremos revisar el 3-itemset: {arroz con leche, coco con merkén, quinoa} Pero antes de calcular su frecuencia, revisamos sus subconjuntos de 2 elementos. Uno de ellos es: {arroz con leche, quinoa} Si ese 2-itemset no fue frecuente, entonces: {arroz con leche, coco con merkén, quinoa} se descarta directamente y no se calcula.
Idea clave: si un subconjunto no es frecuente, el conjunto más grande tampoco puede ser frecuente.

Resumen lógico

1) Generar candidatos 2) Calcular soporte 3) Eliminar los que no cumplen min_support 4) Podar usando subconjuntos no frecuentes 5) Repetir
Esta secuencia resume el funcionamiento general de Apriori.

8. Reducción de dimensionalidad

Aquí conviene entender más la idea general que memorizar fórmulas largas.

PCA

PCA = Principal Component Analysis
Busca resumir la mayor cantidad de varianza posible en menos dimensiones. Crea nuevas variables llamadas componentes principales.

FA

FA = Factor Analysis
Busca factores latentes o comunes que expliquen la relación entre varias variables observadas.

Diferencia conceptual

PCA → resume la variabilidad FA → busca factores ocultos
Esa diferencia suele ser más importante que la fórmula exacta en pruebas teóricas.
