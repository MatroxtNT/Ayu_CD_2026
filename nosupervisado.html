1. ¿Qué es el aprendizaje no supervisado?

El aprendizaje no supervisado es una rama del Machine Learning donde los datos no tienen una variable objetivo conocida. Es decir, el algoritmo no recibe una respuesta correcta para comparar, sino que intenta descubrir estructuras, grupos o relaciones ocultas dentro del conjunto de datos.

Idea principal

Busca patrones sin tener etiquetas conocidas como “aprobado/reprobado” o “fraude/no fraude”.

Usos comunes

Segmentación de clientes, agrupación de documentos, búsqueda de anomalías y reglas de compra frecuente.

Técnicas vistas aquí

Clustering y reglas de asociación, con algoritmos como K-Means, DBSCAN y Apriori.

2. Clustering

El clustering consiste en agrupar observaciones similares en un mismo clúster. La idea es que los datos dentro de un mismo grupo sean parecidos entre sí y diferentes respecto a los de otros grupos.

Ejemplo simple: si tengo datos de clientes, el clustering puede separar grupos como “clientes frecuentes”, “clientes ocasionales” y “clientes de alto gasto”, sin que esas etiquetas hayan sido definidas antes.
En clustering no existe una respuesta correcta previa. El algoritmo descubre agrupaciones según la similitud entre puntos.

3. Métricas de distancia

Muchas técnicas de clustering comparan puntos usando una distancia. Esa distancia indica qué tan cerca o qué tan lejos están dos observaciones en el espacio.

Supongamos dos puntos:
A = (1, 3)
B = (4, 7)

3.1 Distancia Euclidiana

Es la distancia “en línea recta” entre dos puntos. Es la más usada en K-Means.

d(A,B) = √((x2 - x1)^2 + (y2 - y1)^2)

Reemplazando con A = (1,3) y B = (4,7):

d(A,B) = √((4 - 1)^2 + (7 - 3)^2)
d(A,B) = √(3^2 + 4^2)
d(A,B) = √(9 + 16)
d(A,B) = √25
d(A,B) = 5
La distancia euclidiana entre A y B es 5.

3.2 Distancia Manhattan

Suma los desplazamientos horizontales y verticales. También se conoce como distancia de ciudad o city-block.

d(A,B) = |x2 - x1| + |y2 - y1|

Reemplazando con A = (1,3) y B = (4,7):

d(A,B) = |4 - 1| + |7 - 3|
d(A,B) = |3| + |4|
d(A,B) = 3 + 4
d(A,B) = 7
La distancia Manhattan entre A y B es 7.

3.3 Distancia Chebyshev

Considera solo el mayor desplazamiento entre las dimensiones.

d(A,B) = max(|x2 - x1|, |y2 - y1|)

Reemplazando con A = (1,3) y B = (4,7):

d(A,B) = max(|4 - 1|, |7 - 3|)
d(A,B) = max(3, 4)
d(A,B) = 4
La distancia Chebyshev entre A y B es 4.

3.4 Distancia Minkowski

Es una generalización de varias distancias. Según el valor de p, puede comportarse como Manhattan o Euclidiana.

d(A,B) = (|x2 - x1|^p + |y2 - y1|^p)^(1/p)

Si usamos p = 3 y A = (1,3), B = (4,7):

d(A,B) = (|4 - 1|^3 + |7 - 3|^3)^(1/3)
d(A,B) = (3^3 + 4^3)^(1/3)
d(A,B) = (27 + 64)^(1/3)
d(A,B) = 91^(1/3)
d(A,B) ≈ 4.50
La distancia Minkowski con p = 3 entre A y B es aproximadamente 4.50.
Si p = 1, Minkowski se transforma en Manhattan. Si p = 2, se transforma en Euclidiana.

4. K-Means

K-Means es un algoritmo de clustering que agrupa observaciones en K grupos, donde K se define previamente. Su objetivo es que los puntos dentro de un mismo cluster queden lo más cerca posible de su centroide.

K-Means trabaja muy bien cuando los grupos tienen una forma relativamente compacta y cuando las variables ya están en escalas comparables.

4.1 Funcionamiento del algoritmo

  • Se elige el número de clusters K.
  • Se inicializan K centroides.
  • Se calcula la distancia de cada punto a cada centroide, normalmente usando distancia euclidiana.
  • Cada punto se asigna al centroide más cercano.
  • Se recalcula cada centroide como el promedio de los puntos asignados a su cluster.
  • Se repite el proceso hasta que las asignaciones dejen de cambiar o se alcance el máximo de iteraciones.

Fórmula de recálculo del centroide

Si un cluster tiene n puntos, el centroide se obtiene promediando cada coordenada.

C = ( (x1 + x2 + ... + xn) / n , (y1 + y2 + ... + yn) / n )
Ejemplo: si un cluster tiene los puntos (1,3) y (2,4), entonces:
C = ((1+2)/2, (3+4)/2)
C = (1.5, 3.5)

Ejemplo breve de asignación usando distancia euclidiana

Supongamos un punto A = (1,3) y dos centroides:
C1 = (2,4)
C2 = (7,3)

d(A,C1) = √((2 - 1)^2 + (4 - 3)^2) = √2 ≈ 1.41 d(A,C2) = √((7 - 1)^2 + (3 - 3)^2) = √36 = 6
Como la distancia a C1 es menor, el punto A se asigna al cluster del centroide C1.

4.2 Código de ejemplo con Scikit-learn

from sklearn.cluster import KMeans

modelo = KMeans(
    n_clusters=3,
    init="k-means++",
    n_init=10,
    max_iter=300,
    random_state=42
)

modelo.fit(X)

labels = modelo.labels_
centroides = modelo.cluster_centers_

¿Qué significa cada hiperparámetro?

n_clusters

Es la cantidad de clusters K que se quiere formar.

init

Es la forma en que se inicializan los centroides. “k-means++” suele ser una buena opción.

n_init

Es cuántas veces se ejecuta el algoritmo con distintas inicializaciones y se elige la mejor.

max_iter

Es el máximo de iteraciones permitidas para que el algoritmo converja.

random_state

Sirve para fijar una semilla y obtener resultados reproducibles.

5. DBSCAN

DBSCAN es un algoritmo de clustering basado en densidad. En lugar de buscar centroides, identifica regiones donde hay suficientes puntos cercanos entre sí.

DBSCAN es útil cuando los grupos tienen formas irregulares y también cuando se quiere detectar ruido u outliers.

5.1 Funcionamiento del algoritmo

  • El algoritmo toma un punto no visitado.
  • Dibuja alrededor de él una vecindad de radio epsilon (eps).
  • Cuenta cuántos puntos hay dentro de ese radio.
  • Si hay suficientes puntos, el punto se considera core.
  • Si no tiene suficientes vecinos, puede ser border o ruido.
  • Si es core, se crea o expande un cluster usando vecinos conectados por densidad.

Tipos de puntos

Punto core

Es un punto que tiene suficientes vecinos dentro de su radio epsilon. Puede iniciar un cluster y también expandirlo.

Punto border

No cumple MinPts por sí mismo, pero está dentro del radio de un punto core. Pertenece al cluster, pero no lo expande.

Ruido

No cumple MinPts y tampoco está conectado a un punto core. Queda fuera de cualquier cluster.

¿Para qué sirve epsilon?

Epsilon (eps) define el radio de vecindad. Es la distancia máxima para considerar que dos puntos están “cerca”.
Si epsilon es muy pequeño, pueden aparecer muchos puntos como ruido. Si epsilon es muy grande, varios grupos distintos podrían terminar unidos.

¿Qué es MinPts?

MinPts es la cantidad mínima de puntos que debe haber en la vecindad de un punto para que ese punto sea considerado core.

5.2 Código de ejemplo con Scikit-learn

from sklearn.cluster import DBSCAN

modelo = DBSCAN(
    eps=0.5,
    min_samples=5,
    metric="euclidean"
)

labels = modelo.fit_predict(X)

¿Qué significa cada hiperparámetro?

eps

Es el radio de vecindad alrededor de cada punto.

min_samples

Es la cantidad mínima de puntos para considerar a un punto como core.

metric

Es la distancia que se usará para medir cercanía entre puntos, por ejemplo euclidiana.

6. Reglas de asociación

Las reglas de asociación buscan descubrir relaciones frecuentes entre elementos de un conjunto de datos. Se usan mucho en análisis de compras, donde interesa saber qué productos suelen aparecer juntos.

Ejemplo: si muchas veces cuando aparece “Pan” también aparece “Leche”, se puede generar una regla de asociación entre esos productos.

6.1 One-Hot Encoding

Antes de aplicar Apriori, es común transformar las transacciones a formato One-Hot Encoding. Esto consiste en representar cada producto con 1 si aparece en la transacción y 0 si no aparece.

Tabla de transacciones original

Transacción Productos
T1 Pan, Leche
T2 Pan, Mantequilla
T3 Leche, Mantequilla
T4 Pan, Leche, Mantequilla
T5 Pan, Leche

Tabla en One-Hot Encoding

Transacción Pan Leche Mantequilla
T1 1 1 0
T2 1 0 1
T3 0 1 1
T4 1 1 1
T5 1 1 0
Este formato permite contar fácilmente cuántas veces aparece cada ítem o combinación de ítems.

6.2 Algoritmo Apriori

Apriori busca conjuntos de ítems frecuentes llamados itemsets. Parte con ítems individuales y luego construye combinaciones más grandes, filtrando aquellas que no cumplen una frecuencia mínima.

Supongamos min_support = 0.4. Como hay 5 transacciones, una combinación debe aparecer al menos en 2 transacciones para ser considerada frecuente.

Paso 1: calcular soporte de los itemsets de tamaño 1 (k = 1)

Support(Pan) = 4/5 = 0.8 Support(Leche) = 4/5 = 0.8 Support(Mantequilla) = 3/5 = 0.6

Como todos son mayores o iguales a 0.4, los tres pasan a ser itemsets frecuentes de tamaño 1:

{Pan} {Leche} {Mantequilla}

Paso 2: generar itemsets de tamaño 2 (k = 2)

Support(Pan, Leche) = 3/5 = 0.6 Support(Pan, Mantequilla) = 2/5 = 0.4 Support(Leche, Mantequilla) = 2/5 = 0.4

Como todos cumplen el soporte mínimo, también son frecuentes:

{Pan, Leche} {Pan, Mantequilla} {Leche, Mantequilla}

Paso 3: generar itemsets de tamaño 3 (k = 3)

Se combinan los itemsets frecuentes de tamaño 2 para intentar formar uno de tamaño 3:

{Pan, Leche} + {Pan, Mantequilla} → {Pan, Leche, Mantequilla}

Ahora calculamos su soporte:

Support(Pan, Leche, Mantequilla) = 1/5 = 0.2
Como 0.2 es menor que 0.4, ese itemset no es frecuente y se descarta.

6.3 Pruning

El pruning es el proceso de poda usado por Apriori para evitar evaluar combinaciones innecesarias. La idea es simple: si un subconjunto no es frecuente, entonces cualquier conjunto más grande que lo contenga tampoco podrá ser frecuente.

Ejemplo: si {Pan, Mantequilla} no hubiera sido frecuente, entonces no tendría sentido intentar construir {Pan, Leche, Mantequilla}, porque uno de sus subconjuntos ya falló.
Gracias al pruning, Apriori reduce la cantidad de combinaciones a revisar y hace la búsqueda más eficiente.

7. Resumen final

En aprendizaje no supervisado, los algoritmos trabajan sin una variable objetivo conocida. Dentro de este enfoque:

  • Clustering busca agrupar datos similares, como hacen K-Means y DBSCAN.
  • K-Means usa centroides y normalmente distancia euclidiana.
  • DBSCAN usa densidad, epsilon y cantidad mínima de vecinos.
  • Reglas de asociación descubren qué elementos aparecen frecuentemente juntos.
  • Apriori construye itemsets frecuentes paso a paso y elimina combinaciones inviables con pruning.
La idea general es descubrir estructura oculta en los datos, ya sea formando grupos o encontrando relaciones frecuentes entre elementos.
