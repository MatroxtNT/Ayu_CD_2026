1. ¿Qué es el aprendizaje no supervisado?

El aprendizaje no supervisado es una rama del Machine Learning donde los datos no tienen una variable objetivo conocida. El algoritmo no recibe una respuesta correcta para comparar, sino que busca descubrir estructuras, agrupaciones o relaciones ocultas dentro del conjunto de datos.

Idea principal

Busca patrones sin etiquetas conocidas como “aprobado/reprobado” o “fraude/no fraude”.

Usos comunes

Segmentación de clientes, agrupación de documentos, búsqueda de anomalías y reglas de compra frecuente.

Enfoques vistos

Clustering, reglas de asociación y reducción de la dimensionalidad.

3. Categoría: Clustering

El clustering consiste en agrupar observaciones similares en un mismo clúster. La idea es que los datos dentro de un mismo grupo sean parecidos entre sí y diferentes respecto a los de otros grupos.

Ejemplo simple: si tengo datos de clientes, el clustering puede separar grupos como “clientes frecuentes”, “clientes ocasionales” y “clientes de alto gasto”, sin que esas etiquetas hayan sido definidas antes.
En clustering no existe una respuesta correcta previa. El algoritmo descubre agrupaciones según la similitud entre puntos.

3.1 Métricas de distancia

Base matemática del clustering

Muchas técnicas de clustering comparan puntos usando una distancia. Esa distancia indica qué tan cerca o qué tan lejos están dos observaciones en el espacio.

Supongamos dos puntos:
A = (1, 3)
B = (4, 7)

Distancia Euclidiana

Es la distancia “en línea recta” entre dos puntos. Es la más usada en K-Means.

d(A,B) = √((x2 - x1)^2 + (y2 - y1)^2)
d(A,B) = √((4 - 1)^2 + (7 - 3)^2)
d(A,B) = √(3^2 + 4^2)
d(A,B) = √25 = 5

Distancia Manhattan

Suma los desplazamientos horizontales y verticales.

d(A,B) = |x2 - x1| + |y2 - y1|
d(A,B) = |4 - 1| + |7 - 3|
d(A,B) = 3 + 4 = 7

Distancia Chebyshev

Considera solo el mayor desplazamiento entre dimensiones.

d(A,B) = max(|x2 - x1|, |y2 - y1|)
d(A,B) = max(3,4) = 4

Distancia Minkowski

Generaliza varias distancias según el valor de p.

d(A,B) = (|x2 - x1|^p + |y2 - y1|^p)^(1/p)
Si p = 3:
d(A,B) = (3^3 + 4^3)^(1/3)
d(A,B) = 91^(1/3) ≈ 4.50
Si p = 1, Minkowski se transforma en Manhattan. Si p = 2, se transforma en Euclidiana.

3.2 K-Means

Clustering basado en centroides

K-Means agrupa observaciones en K grupos, donde K se define previamente. Su objetivo es que los puntos dentro de un mismo cluster queden lo más cerca posible de su centroide.

K-Means funciona especialmente bien cuando los grupos tienen forma relativamente compacta y cuando las variables están en escalas comparables.

Funcionamiento del algoritmo

  • Se elige el número de clusters K.
  • Se inicializan K centroides.
  • Se calcula la distancia de cada punto a cada centroide, usualmente con distancia euclidiana.
  • Cada punto se asigna al centroide más cercano.
  • Se recalcula cada centroide como el promedio de los puntos asignados.
  • Se repite el proceso hasta que deje de cambiar la asignación o se alcance el máximo de iteraciones.

Fórmula de recálculo del centroide

C = ( (x1 + x2 + ... + xn) / n , (y1 + y2 + ... + yn) / n )
Si un cluster contiene los puntos (1,3) y (2,4), su centroide es:
C = ((1+2)/2, (3+4)/2)
C = (1.5, 3.5)

Ejemplo breve con distancia euclidiana

A = (1,3) C1 = (2,4) C2 = (7,3) d(A,C1) = √((2 - 1)^2 + (4 - 3)^2) = √2 ≈ 1.41 d(A,C2) = √((7 - 1)^2 + (3 - 3)^2) = 6
Como la distancia a C1 es menor, el punto A se asigna al cluster del centroide C1.

Código de ejemplo con Scikit-learn

from sklearn.cluster import KMeans

modelo = KMeans(
    n_clusters=3,
    init="k-means++",
    n_init=10,
    max_iter=300,
    random_state=42
)

modelo.fit(X)

labels = modelo.labels_
centroides = modelo.cluster_centers_

n_clusters

Cantidad de clusters K que se quiere formar.

init

Forma de inicializar los centroides. “k-means++” suele ser una buena opción.

n_init

Cantidad de inicializaciones probadas antes de elegir la mejor.

max_iter

Máximo de iteraciones permitidas para converger.

random_state

Sirve para fijar una semilla y hacer reproducible el resultado.

3.3 DBSCAN

Clustering basado en densidad

DBSCAN es un algoritmo de clustering basado en densidad. En lugar de buscar centroides, identifica regiones donde hay suficientes puntos cercanos entre sí.

DBSCAN es útil cuando los grupos tienen formas irregulares y también cuando se quiere detectar ruido u outliers.

Funcionamiento del algoritmo

  • Toma un punto no visitado.
  • Dibuja alrededor de él una vecindad de radio epsilon (eps).
  • Cuenta cuántos puntos hay dentro de ese radio.
  • Si hay suficientes puntos, el punto se considera core.
  • Si no, puede ser border o ruido.
  • Si es core, se crea o expande un cluster usando vecinos conectados por densidad.

Punto core

Tiene suficientes vecinos dentro de su radio epsilon. Puede iniciar y expandir un cluster.

Punto border

No cumple MinPts por sí mismo, pero está dentro del radio de un punto core.

Ruido

No cumple MinPts y tampoco está conectado a un punto core.

Epsilon (eps) define el radio de vecindad. Es la distancia máxima para considerar que dos puntos están cerca.
Si epsilon es muy pequeño, aparecen muchos puntos como ruido. Si es muy grande, varios clusters distintos pueden unirse.

Código de ejemplo con Scikit-learn

from sklearn.cluster import DBSCAN

modelo = DBSCAN(
    eps=0.5,
    min_samples=5,
    metric="euclidean"
)

labels = modelo.fit_predict(X)

eps

Radio de vecindad alrededor de cada punto.

min_samples

Cantidad mínima de puntos para considerar a un punto como core.

metric

Distancia usada para medir cercanía entre puntos, por ejemplo euclidiana.

3.4 Clustering jerárquico

Clustering por niveles

El clustering jerárquico construye una estructura en forma de árbol llamada dendrograma. Su idea es unir observaciones parecidas paso a paso o dividir grandes grupos en subgrupos más pequeños.

Enfoque aglomerativo

Comienza con cada punto como un cluster individual y luego va fusionando los más similares.

Enfoque divisivo

Comienza con un gran grupo y va separándolo en subgrupos.

Ventaja

No obliga a fijar desde el inicio una sola partición; el dendrograma muestra varios niveles de agrupación.

A diferencia de K-Means, aquí la estructura jerárquica del agrupamiento se puede visualizar de forma más natural.

3.5 Gradient Boosting

Aclaración importante

Gradient Boosting no corresponde a clustering ni a aprendizaje no supervisado. En realidad es un método de aprendizaje supervisado usado en clasificación y regresión.

Se incluye aquí solo como aclaración estructural, porque a veces se confunde con otros algoritmos, pero no forma parte de las técnicas de clustering.
Su lógica general es combinar varios modelos débiles, usualmente árboles pequeños, para corregir errores progresivamente y construir un modelo final más preciso.

4. Categoría: Reglas de asociación

Las reglas de asociación buscan descubrir relaciones frecuentes entre elementos de un conjunto de datos. Se usan mucho en análisis de compras, donde interesa saber qué productos suelen aparecer juntos.

Ejemplo: si muchas veces cuando aparece “Pan” también aparece “Leche”, se puede generar una regla de asociación entre esos productos.
Una regla como Pan → Leche indica co-ocurrencia, no causalidad.

4.1 Probabilidad básica, conjunta, condicional y Bayes

Supongamos que en una población se quiere estudiar una enfermedad mediante un test.

Se sabe lo siguiente:

  • El 1% de las personas tiene la enfermedad.
  • Si una persona está enferma, el test sale positivo en el 90% de los casos.
  • En total, el test sale positivo en el 5% de las personas.

Probabilidad básica

P(A) = casos favorables / casos totales

Probabilidad conjunta

P(A ∩ B)

Probabilidad condicional

P(B|A) = P(A ∩ B) / P(A)

Teorema de Bayes

P(A|B) = (P(B|A) * P(A)) / P(B)
Estas ideas sirven como base para interpretar soporte, confidence y otras métricas de reglas de asociación.

4.2 One-Hot Encoding

Antes de aplicar Apriori, es común transformar las transacciones a formato One-Hot Encoding. Esto consiste en representar cada producto con 1 si aparece en la transacción y 0 si no aparece.

Tabla de transacciones original

Transacción Productos
T1 Pan, Leche
T2 Pan, Mantequilla
T3 Leche, Mantequilla
T4 Pan, Leche, Mantequilla
T5 Pan, Leche

Tabla en One-Hot Encoding

Transacción Pan Leche Mantequilla
T1 1 1 0
T2 1 0 1
T3 0 1 1
T4 1 1 1
T5 1 1 0
Este formato permite contar fácilmente cuántas veces aparece cada ítem o combinación de ítems.

4.3 Algoritmo Apriori

Apriori busca conjuntos de ítems frecuentes llamados itemsets. Parte con ítems individuales y luego construye combinaciones más grandes, filtrando las que no cumplen un soporte mínimo.

Supongamos min_support = 0.4. Como hay 5 transacciones, una combinación debe aparecer al menos en 2 transacciones para ser frecuente.

Paso 1: itemsets de tamaño 1 (k = 1)

Support(Pan) = 4/5 = 0.8 Support(Leche) = 4/5 = 0.8 Support(Mantequilla) = 3/5 = 0.6
{Pan} {Leche} {Mantequilla}

Paso 2: itemsets de tamaño 2 (k = 2)

Support(Pan, Leche) = 3/5 = 0.6 Support(Pan, Mantequilla) = 2/5 = 0.4 Support(Leche, Mantequilla) = 2/5 = 0.4
{Pan, Leche} {Pan, Mantequilla} {Leche, Mantequilla}

Paso 3: itemsets de tamaño 3 (k = 3)

{Pan, Leche} + {Pan, Mantequilla} → {Pan, Leche, Mantequilla} Support(Pan, Leche, Mantequilla) = 1/5 = 0.2
Como 0.2 es menor que 0.4, ese itemset no es frecuente y se descarta.

4.4 Métricas de validación / Resultados para Pan y Leche

Métrica Fórmula Interpretación
Support Support(Pan) = P(Pan) = (4/5) = 0.8
Support(Pan → Leche) = P(Pan ∩ Leche) = (3/5) = 0.6 		Frecuencia dentro del dataset
Confidence Confidence(Pan → Leche) = Support(Pan → Leche) / Support(Pan)
Confidence = (0.6 / 0.8) = 0.75 		“Leche ocurre frecuentemente cuando ocurre Pan”
Lift Lift(Pan → Leche) = Support(Pan → Leche) / (Support(Pan) * Support(Leche))
Lift = (0.6 / (0.8 * 0.8)) = 0.9375 		Relación negativa leve
Leverage Leverage(Pan → Leche) = Support(Pan → Leche) - (Support(Pan) * Support(Leche))
Leverage(Pan → Leche) = (0.6 - (0.8 * 0.8)) = -0.04 		Ocurren menos de lo esperado
Conviction Conviction(Pan → Leche) = (1 - Support(Leche)) / (1 - Confidence(Pan → Leche))
Conviction(Pan → Leche) = (1 - 0.8) / (1 - 0.75) = 0.8 		La regla no es fuerte y está cercana a independencia
En este ejemplo, Pan y Leche sí aparecen juntos varias veces, pero las métricas muestran que la relación no es especialmente fuerte.

4.5 Pruning

El pruning es el proceso de poda usado por Apriori para evitar evaluar combinaciones innecesarias. Si un subconjunto no es frecuente, entonces cualquier conjunto más grande que lo contenga tampoco podrá ser frecuente.

Si {Pan, Mantequilla} no hubiera sido frecuente, entonces no tendría sentido intentar construir {Pan, Leche, Mantequilla}, porque uno de sus subconjuntos ya falló.
Gracias al pruning, Apriori reduce la cantidad de combinaciones a revisar y hace la búsqueda más eficiente.

5. Categoría: Reducción de dimensionalidad

La reducción de dimensionalidad es una técnica que busca disminuir la cantidad de variables de un conjunto de datos, intentando conservar la mayor cantidad posible de información importante.

Cada atributo puede entenderse como una dimensión. Si un dataset tiene muchas columnas, trabajar con todas puede ser difícil, costoso o redundante.

5.1 PCA

PCA o Principal Component Analysis transforma las variables originales en nuevas variables llamadas componentes principales.

Estos componentes buscan capturar la mayor varianza posible de los datos usando menos dimensiones.

PCA no elimina simplemente variables: crea nuevas dimensiones que resumen la información original.
Si varias variables están muy relacionadas entre sí, PCA puede condensar esa información en pocos componentes principales.

5.2 FA

FA o Factor Analysis busca encontrar factores latentes que expliquen la relación entre varias variables observadas.

A diferencia de PCA, FA se enfoca más en la estructura común entre variables que en conservar la varianza total.

FA intenta responder qué factores ocultos podrían estar detrás de las variables medidas.
En una encuesta, varias preguntas podrían estar reflejando un mismo factor latente, como satisfacción, estrés o compromiso.
Diferencia clave: PCA resume la mayor variabilidad posible, mientras que FA busca factores comunes o latentes que expliquen la relación entre variables.

6. Resumen final

  • Clustering agrupa observaciones similares y aquí se revisaron K-Means, DBSCAN y clustering jerárquico.
  • Reglas de asociación estudian relaciones frecuentes entre ítems usando One-Hot Encoding, Apriori y métricas como support, confidence, lift, leverage y conviction.
  • Reducción de dimensionalidad busca trabajar con menos variables, y aquí se diferenciaron PCA y FA.
  • Gradient Boosting se dejó como aclaración porque es supervisado y no pertenece a no supervisado.
La estructura correcta ayuda a entender que dentro del tema general de aprendizaje no supervisado existen categorías diferentes, cada una con un propósito propio.
